分布式Linux性能分析监控系统

Created2025-03-22|Updated2025-03-25
|Post Views:

项目简介

本项目是一个分布式Linux性能分析监控系统，旨在实时监控和分析服务器的性能状态（如CPU、内存、网络、软中断等），并通过UI界面展示数据。系统采用分布式架构，通过gRPC/自研RPC实现服务端与客户端的通信，使用Docker容器化部署环境，支持快速在多台服务器上部署和运行。

适合岗位：Linux C++开发、通用C++后端开发等。

技术栈

  • ​编程语言：C++、shell
  • ​核心特性：C++常用特性（如多线程、智能指针、STL等）
  • ​网络编程：基于事件驱动、非阻塞 I/O 和 Reactor 模式
  • ​容器化：Docker
  • ​通信框架：gRPC、自研RPC
  • ​数据序列化：Protobuf
  • ​构建工具：CMake
  • ​UI框架：Qt
  • ​代码管理：Git、clang-format

项目模块详解

1. Docker模块

  • ​功能：通过Dockerfile指定CMake、gRPC、Protobuf等依赖，构建整个项目环境。
  • ​挑战：解决多台服务器环境不一致的问题，确保项目快速部署。
  • ​成果：将部署时间从数小时缩短至10分钟以内。

2. Monitor模块

  • ​功能：实时监控服务器的CPU状态、系统负载、软中断、内存和网络等性能指标。
  • ​挑战：设计高扩展性的监控框架，支持动态添加新的监控指标。
  • ​成果：通过工厂模式实现模块化设计，减少后续新增监控指标开发时间。

3. 通信模块

3.1 GRPC通信模块

  • ​功能：通过gRPC框架构建服务端（Server）和客户端（Client）。
  • ​实现：
    • 服务端部署在监控服务器上，负责采集性能数据。
    • 客户端生成库，供Monitor模块和Display模块调用。
  • ​特点：
    • 模块间通过gRPC服务通信，降低耦合性。
    • 支持分布式部署，便于扩展。

3.2 自研RPC通信模块

功能
  • 实现一个轻量级、高性能的 RPC 框架，支持分布式系统中的服务调用。
  • 基于 Protobuf 定义数据结构和 RPC 接口，提供高效的序列化和反序列化。
  • 采用多 Reactor 多线程模型，支持高并发通信。
实现

  1. ​通信协议：

    • 基于 TCP 协议，自定义二进制协议，实现高效数据传输。
    • 使用 Protobuf 定义数据结构和 RPC 接口，确保数据的高效序列化和反序列化。

  2. ​服务端：

    • 采用多 Reactor 多线程模型，支持并发处理客户端请求。
    • 通过回调机制处理网络事件，提高系统响应速度。
    • 提供服务注册与发现功能，支持分布式系统中的服务调用。

  3. ​客户端：

    • 封装 Socket 接口，实现与服务端的通信。
    • 通过 Protobuf 动态生成 RPC 接口代码，减少开发工作量。
    • 支持异步调用，提高系统吞吐量。

  4. ​网络模块：

    • 基于事件驱动、非阻塞 I/O 和 Reactor 模式，实现高性能网络通信。
    • 使用多线程模型，支持高并发通信。

4. Protobuf数据序列化

  • ​功能：使用Protobuf定义整个项目的数据结构。
  • ​特点：
    • 高效的数据序列化和反序列化。
    • 数据结构清晰，易于维护和扩展。

5. Display模块

  • ​功能：通过UI界面展示监控数据。
  • ​实现：
    • ​UI界面：使用Qt框架构建，包括QWidget、QTableView、QStackedLayout、QPushButton等组件。
    • ​DataModel：通过继承QAbstractTableModel，构建cpu_model、softirq_model、mem_model等，每3秒刷新一次数据。
  • ​特点：
    • 界面简洁，数据展示直观。
    • 支持实时刷新，动态展示服务器性能状态。
